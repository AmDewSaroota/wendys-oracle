📋 วิธีวิเคราะห์ข้อมูล (Methodology)

▼

1. การเก็บข้อมูล

  • เซนเซอร์: MT15 (PM2.5, PM10, CO2, CO, HCHO, TVOC, Temp, Humidity, AQI)
  • ความถี่: ทุก 5 นาที ตลอดช่วงหุงต้ม (~2 ชม./session)
  • 10 ครัวเรือน · 2 session/วัน · เตาดั้งเดิม + เตาชีวมวล

2. การหักค่าแวดล้อม (Ambient Subtraction)

  • ค่าเฉลี่ย 10 นาทีแรก ของแต่ละ session = ค่าแวดล้อม (Ambient)
  • ค่าสุทธิ (Net) = ค่าที่วัดได้ − Ambient ของ session นั้น
  • ค่าติดลบถูกปรับเป็น 0 · Readings ช่วง Ambient ไม่นำมาวิเคราะห์

3. ข้อจำกัด

  • เก็บข้อมูลคนละช่วงเวลา (ไม่ใช่ paired design)
  • Semiconductor sensor มีความคลาดเคลื่อนมากกว่า gravimetric method
  • MT15 ส่ง CO/TVOC ขึ้น Cloud อัตโนมัติ

🏠 ภาพรวมระบบ ▼

ระบบ Biomass Stove Monitoring ใช้เซนเซอร์วัดคุณภาพอากาศเปรียบเทียบระหว่าง เตาชีวมวล (eco) กับ เตาดั้งเดิม (old)

การไหลของข้อมูล:

📡 เซนเซอร์ MT15 → ☁️ Tuya Cloud → ⚙️ Sync API (ทุก 5 นาที) → 🗄️ ฐานข้อมูล → 📊 Dashboard

คำศัพท์สำคัญ:

  • Session (เซสชัน) — รอบการเก็บข้อมูล 1 ครั้ง ใช้เวลา 130 นาที (2 ชม. 10 นาที) ระบบบันทึกค่าทุก 5 นาที ได้สูงสุด ~26 readings/session — ถ้าได้ ≥ 22 readings = สมบูรณ์ ✅ ถ้าน้อยกว่า = ไม่สมบูรณ์ ⚠️ (ข้อมูลยังใช้ได้ แต่อาจไม่ครอบคลุมทั้ง session)
  • Baseline — ค่าอากาศปกติก่อนจุดเตา (10 นาทีแรก) ใช้หักล้างค่าพื้นฐานออก ถ้าอาสาเลือก "เก็บ TVOC/CO" ระบบจะให้กรอกค่าตั้งแต่ช่วง Baseline เพื่อเปรียบเทียบก่อน-หลังจุดเตา
  • ช่วงเก็บข้อมูล (Collection Period) — กำหนดว่าแต่ละบ้านใช้เตาอะไร (เตาชีวมวล/เตาดั้งเดิม) ในช่วงเวลาใด
  • Cooldown — ช่วงพักเครื่อง 5 ชม. ระหว่างเซสชัน

หน้าจอหลัก 3 ส่วน:

  • ข้อมูลเบื้องต้น — ภาพรวมของทุกเซนเซอร์ สรุปรายวัน แผนที่
  • ข้อมูลเชิงลึก — วิเคราะห์ session, เปรียบเทียบ eco vs old, กราฟ
  • จัดการระบบ (แอดมิน) — ตั้งค่าบ้าน อุปกรณ์ ช่วงเก็บข้อมูล อนุมัติ

ลำดับการตั้งค่าเริ่มต้น:

📁 โครงการ → 👤 อาสาสมัคร → 🏠 บ้าน → 📡 เซนเซอร์ → 📅 ช่วงเก็บข้อมูล

ต้องกรอกเรียงลำดับ เพราะข้อมูลเชื่อมกัน (บ้านต้องมีโครงการ+อาสาก่อน ฯลฯ)

🔥 ขั้นตอนที่ 4: Baseline + เลือกกรอก TVOC/CO ▼

Baseline (10 นาทีแรก):

ระบบเก็บค่าอากาศปกติอัตโนมัติ ~10 นาที เพื่อเป็นค่าฐาน (ค่าอากาศแวดล้อมก่อนจุดเตา)

ระบบจะถามว่า "วันนี้ต้องกรอกค่า TVOC/CO มั้ย?"

✅ ตอบ "เก็บ"
  • • ทุก 5 นาที จะมีแจ้งเตือนให้กรอกค่า (ต้องเปิดหน้าจอทิ้งไว้จึงจะแจ้งเตือนได้)
  • • อ่านค่า TVOC (mg/m³) และ CO (ppm) จากหน้าจอเซนเซอร์
  • • กรอกทั้งสองค่า → กด "บันทึก"
  • • ต้องอยู่ใกล้เซนเซอร์ตลอด
❌ ตอบ "ไม่เก็บ"
  • • ไม่ต้องกรอกอะไร
  • • เซนเซอร์เก็บค่า PM2.5, CO2 ฯลฯ อัตโนมัติ
  • • ปิดหน้าจอไปเลยก็ได้ หลังกดเริ่มจุดเตา
  • • ⚠️ มือถือที่ปล่อย Hotspot ต้องวางไว้ใกล้เซนเซอร์ และห้ามปิด Hotspot ตลอดการเก็บข้อมูล

เมื่อ Baseline พร้อม:

จะปรากฏปุ่มสีแดงใหญ่ "🔥 เริ่มจุดเตาแล้ว" — จุดเตาจริง แล้วค่อยกดปุ่มนี้

⏱️ ขั้นตอนที่ 5: ระหว่างเก็บข้อมูล (130 นาที) ▼
  • ระบบเก็บข้อมูลจากเซนเซอร์ทุก 5 นาทีผ่าน Cloud อัตโนมัติ
  • แถบ progress จะแสดงเวลาที่ผ่านไป เช่น "65 นาที / 130 นาที"
  • คาดว่าจะได้ ~27 ค่า ต่อเซสชัน
  • เซสชันจะ ปิดอัตโนมัติ เมื่อครบ 130 นาที
💡 ถ้าไม่ได้กรอก TVOC: หลังกด "เริ่มจุดเตาแล้ว" สามารถปิดหน้าจอไปเลยได้ เซนเซอร์ + Cloud ทำงานเองอัตโนมัติ แค่อย่าปิด Hotspot!
⚠️ อย่าปิด Hotspot! เซนเซอร์ต้องใช้ WiFi จาก Hotspot ส่งข้อมูลขึ้น Cloud ตลอด ถ้าปิด Hotspot เซนเซอร์จะ offline และข้อมูลจะขาดช่วง
❓ สถานการณ์พิเศษ ▼

🏠 บ้านซ้ำ — อาสาเปิดหน้าเก็บข้อมูลบ้านที่มีคนเก็บอยู่แล้ว

ระบบจะแสดง "🚫 มีอาสาคนอื่นกำลังเก็บข้อมูลบ้านนี้อยู่" และบล็อคไม่ให้เข้า

ล็อคจะหมดอายุใน 3 นาที หลังมือถือเครื่องเดิมหยุดตอบ (ปิดหน้าจอ/แบทหมด/ปิดแอป)

💡 ถ้าเป็นคนเดียวกันแต่กด refresh: ระบบจำ browser ได้ → ปล่อยผ่านเลย ไม่ถูกบล็อค

🔋 เปลี่ยนมือถือระหว่างเก็บข้อมูล (แบทหมด / เครื่องค้าง)

1
เปิด Hotspot บนมือถือเครื่องใหม่ (ชื่อ Biomass / รหัส Biomass2026)
2
Scan QR บนสติกเกอร์เซนเซอร์ด้วยมือถือเครื่องใหม่
3
รอ ~3 นาที → ระบบจะตรวจพบว่าเครื่องเดิมไม่ตอบ แล้วปลดล็อคให้เครื่องใหม่เข้าได้
4
เลือกบ้านเดิม → ระบบจะต่อ session เดิมให้อัตโนมัติ พร้อมโหลดค่า TVOC/CO ที่กรอกไว้กลับมา
⚠️ ข้อมูลเซนเซอร์ไม่หาย — เซนเซอร์ส่งข้อมูลขึ้น Cloud ผ่าน Hotspot ตลอด แม้เปลี่ยนมือถือ session ยังเก็บข้อมูลต่อเนื่อง แค่ต้องมี Hotspot ชื่อเดิมเปิดอยู่
