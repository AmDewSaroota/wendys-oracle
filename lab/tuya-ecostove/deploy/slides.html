Baseline แยกไม่ได้ — ทำไมระบบเดิมไม่เพียงพอ

🔴 สาเหตุที่ 1 — ไม่รู้จังหวะ "เริ่มจุดเตา"
ระบบเก็บข้อมูลตลอดเวลาที่เซนเซอร์ออนไลน์ แต่ไม่รู้ว่าตอนไหนจุดเตาจริง → ข้อมูล ambient กับ cooking ปนกัน
🔴 สาเหตุที่ 2 — Baseline อัตโนมัติไม่น่าเชื่อถือ
ใช้ 10 นาทีแรกเป็น baseline → แต่ถ้าจุดเตาไปก่อนเซนเซอร์ออนไลน์ baseline จะปนควัน
🟡 สาเหตุที่ 3 — เซนเซอร์ออนไลน์ตลอด (WiFi ถาวร)
ถ้าบ้านมี WiFi → เซนเซอร์ต่อ 24 ชม. → ไม่รู้ว่า session ไหนมีการใช้เตาจริง
🔴 สาเหตุที่ 4 — ไม่มีระบบแจ้งว่าเซนเซอร์ออนไลน์
ผู้ใช้เตาไม่รู้ว่าเซนเซอร์เชื่อมต่อ WiFi สำเร็จหรือยัง → อาจเริ่มจุดเตาก่อนเซนเซอร์พร้อม → baseline เก็บไม่ครบ

✅ สิ่งที่ต้องการ

เก็บ baseline (อากาศปกติ) → จุดเตา → กดปุ่ม → เก็บค่าจริง
✅ แยกข้อมูลชัดเจน → หักลบ baseline ได้แม่นยำ

⚠️ สิ่งที่เกิดจริงในระบบเดิม

เซนเซอร์ online → จุดเตาแล้วก่อน? → ไม่รู้ → baseline ผิด
❌ ข้อมูลปนกัน → ผลเปรียบเทียบไม่น่าเชื่อถือ
💡 ทางออก
ต้องมีคนจริงๆอยู่ที่บ้าน คอยบอกระบบว่า "เริ่มจุดเตาแล้ว" → ระบบอาสาสมัคร

ระบบอาสาสมัคร — แอปเก็บข้อมูลบนมือถือ

ออกแบบเฉพาะสำหรับอาสา — ใช้งานง่ายบนมือถือ ไม่ต้องสมัคร ไม่ต้อง login

📍

GPS ตรวจจับบ้าน

  • เปิดหน้าเว็บ → ขอตำแหน่ง GPS
  • คำนวณระยะกับพิกัดบ้าน 10 หลัง
  • < 30 ม. → ยืนยันอัตโนมัติ
  • 30–200 ม. → ให้กดยืนยัน
  • > 200 ม. → เลือกบ้านเอง (ป้องกันบ้านซ้ำ)
🔥

ปุ่ม "เริ่มจุดเตา"

  • กดเมื่อเริ่มจุดเตา
  • ระบบตัด baseline ทันที
  • บันทึกเวลาแม่นยำ
  • ได้ baseline ที่เชื่อถือได้
📝

บันทึก TVOC

  • Tuya Cloud ส่งค่า TVOC ไม่ได้
  • อาสาอ่านจากหน้าจอเซนเซอร์
  • กรอกค่าในแอป ทุก 5 นาที
  • 🔔 มี Noti เตือน (เสียง+สั่น+แจ้งเตือน)
  • ⚠️ เวิร์คเมื่อเปิดหน้าเว็บค้างไว้ — ปิดจอ/ล็อคเครื่องจะไม่ดัง
🛡️

ป้องกันข้อมูลผิดพลาด

  • จำกัดรัศมี 200 ม. จาก GPS
  • แสดงสถานะเซนเซอร์ real-time
  • Timestamp ทุก action (จุดเตา, TVOC, จบ)
  • Dashboard ดูย้อนหลังเป็น session
📎 QR Code แปะที่เซนเซอร์ — อาสาสแกนเข้าใช้งานได้เลย ไม่ต้องจำ URL  |  🔗 หรือเปิดผ่าน Dashboard ก็ได้

👥 อาสาสมัคร

  • ใครเป็นอาสา? — คนในบ้านเอง หรือคนนอกมาช่วย?
  • อาสา 10 คนมี มือถือ smartphone มั้ย?
  • มือถือ dual-SIM หรือ ซิมเดียว?
  • ถ้าไม่มีมือถือ → งบจัดหาจากไหน? (~1,500–2,500 ฿/เครื่อง)
2️⃣

📡 อินเทอร์เน็ต

  • เลือก แผน A / B หรือ mix? (ไม่แนะนำ C)
  • 10 บ้านอยู่ใกล้กันแค่ไหน? (Pocket WiFi แชร์ได้มั้ย)
  • บ้านไม้หรือคอนกรีต? (มีผลต่อสัญญาณ WiFi)
3️⃣

🏠 บ้านตัวอย่าง

  • 10 บ้าน — 3 เดือนแรกเตาเก่า → 3 เดือนหลังเตาใหม่
  • ต้องการพิกัด GPS ทุกหลัง (ลงทะเบียนในระบบ)
  • Baseline เก็บนานเท่าไหร่? (5 / 15 / 30 นาที?)
4️⃣

📅 ตารางเก็บข้อมูล ✅ ตั้งค่าได้แล้ว

  • เลือกวันเก็บประจำสัปดาห์ได้ (default จ-ศ / หยุด ส-อา)
  • 2 session/วัน — สูงสุด 2 ครั้ง × 2 ชม. 10 นาที
  • เพิ่มวันหยุดพิเศษได้ (สงกรานต์ ฯลฯ)
  • Quiet Hours ช่วงกลางคืน — ไม่เปลือง API
  • ปฏิทินเดือน + แสดง Projected Quota ให้เห็นชัด
