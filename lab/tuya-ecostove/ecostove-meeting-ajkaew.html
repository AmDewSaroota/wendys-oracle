✅ ทำแล้ว — Offline Handling

ถ้าเน็ตหลุด / ไฟดับ — ระบบจัดการอย่างไร?

✅ สถานการณ์ 1: เน็ตหลุดแป๊บเดียว (<10 นาที)
กำลังเก็บข้อมูล → เน็ตหลุด 5 นาที → เน็ตกลับมา → ต่อ session เดิม
ผลลัพธ์: ข้อมูลขาดแค่ช่วงที่หลุด → session สมบูรณ์
⚠️ สถานการณ์ 2: เน็ตหลุดนาน (>10 นาที)
กำลังเก็บข้อมูล → เน็ตหลุด 20 นาที → ยกเลิก session → เริ่ม session ใหม่
ผลลัพธ์: Tag "incomplete" → ไม่นับเป็น 1 ใน 2 โควต้า → เริ่ม session ใหม่ได้เมื่อเน็ตกลับ
⚡ สถานการณ์ 3: ไฟดับที่บ้าน
เซนเซอร์มีแบต → ยังทำงาน → แต่เราเตอร์ดับ → เน็ตหลุด → ไฟมา = เราเตอร์กลับมาทำงาน
ผลลัพธ์: เซนเซอร์ไม่ดับ แต่ส่งข้อมูลไม่ได้ช่วงเราเตอร์ดับ → ข้อมูลเดิมปลอดภัยใน DB
📱 สถานการณ์ 4: อาสาลืมเปิด Hotspot
ถึงเวลาเก็บข้อมูล → ไม่มีเน็ต → ไม่มีข้อมูลรอบนี้
ผลลัพธ์: ขาดข้อมูลรอบนั้น → LINE แจ้งเตือนอาสา (ฟีเจอร์นี้ต้องอัพเป็น Light Plan ฿400/เดือน)
🖥️ สถานการณ์ 5: Server ล่ม
Vercel Cron ทำงาน → Cron ล้มเหลว → Vercel retry อัตโนมัติ
ผลลัพธ์: Serverless = ไม่มี server ล่ม — ข้อมูลเดิมปลอดภัยใน DB
🛡️

หลักการสำคัญ

  • ข้อมูลเดิมไม่มีวันหาย — เก็บใน DB แยก
  • ระบบกลับมาเอง — Serverless ไม่ต้องดูแล
  • ขาดบ้างไม่เป็นไร — วิเคราะห์จากที่มี
  • มีปัญหาเรื่องระบบ → ติดต่อเรา
