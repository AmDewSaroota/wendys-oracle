ระบบค้างระหว่างใช้งาน — บันทึกข้อมูลไม่ได้

เกิดอะไรขึ้น?

ระบบเดิมใช้ Supabase Authentication ซึ่งออก"ตั๋วเข้าใช้งาน" (JWT Token) ให้ทุกครั้งที่ล็อคอิน

ตั๋วนี้มีวันหมดอายุ — พอหมดอายุระหว่างใช้งาน ทั้งระบบจะค้าง

ไม่ใช่แค่เขียนข้อมูลไม่ได้ แม้แต่การอ่านข้อมูลก็พังไปด้วย เพราะตั๋วเสียทำพิษกับทุก request

แก้มากี่รอบแล้ว?

❌ รอบ 1 — เพิ่ม auto-refresh token → ยังพัง
❌ รอบ 2 — ลบ token เก่าอัตโนมัติ → ยังพัง
❌ รอบ 3 — retry เมื่อ token เสีย → ยังพัง
ปัญหาอยู่ที่ตัวระบบ Supabase Auth เอง
ไม่ใช่โค้ดของเรา — แก้จากฝั่งเราไม่ได้

ระบบเดิม vs ระบบใหม่

❌ ระบบเดิม (Supabase Auth)

ออกแบบมาสำหรับแอปขนาดใหญ่ (ผู้ใช้หลักพัน-หมื่น)
ล็อคอินด้วย email + password ส่วนตัว
ออกตั๋ว JWT ที่ หมดอายุทุกชั่วโมง
ตั๋วหมดอายุ → ทั้งเว็บพัง
ลืมรหัส → ต้องรีเซ็ตผ่าน email
ซับซ้อนเกินความจำเป็นสำหรับทีมเล็ก

✅ ระบบใหม่ (PIN ส่วนตัว)

ออกแบบให้เหมาะกับทีมเล็ก (10-15 คน)
ล็อคอินด้วย ชื่อ + PIN 4 หลัก
ไม่มีตั๋ว ไม่มีวันหมดอายุ
ใช้งานได้ตลอด ไม่มีค้าง
ลืม PIN → แอดมินรีเซ็ตให้ต่อหน้า
เรียบง่าย เข้าใจง่าย ใช้ง่าย

ระบบป้องกัน 2 ชั้น

🔒

ชั้นที่ 1 — ผู้ใช้งานทั่วไป

→ ล็อคอินด้วย ชื่อ + PIN ส่วนตัว
→ เพิ่ม/แก้ไขข้อมูล (บ้าน, เซนเซอร์, อาสา)
→ ดูรายงานและส่งออกข้อมูล
ทำไม่ได้: เปลี่ยนรหัสสมัคร, ลบแอดมินคนอื่น, เข้าตั้งค่าระบบ
🔐

ชั้นที่ 2 — Super Admin

→ ต้องใส่ รหัส Super Admin เพิ่มอีกชั้น
→ เปลี่ยนรหัสสมัคร (เมื่อนักศึกษาจบ)
→ ลบแอดมินออกจากระบบ
→ รีเซ็ต PIN ให้คนที่ลืม
เฉพาะ: อาจารย์ หรือผู้ดูแลระบบหลักเท่านั้น

FAQ — เผื่ออาจารย์ถาม

ปลอดภัยไหม?
ปลอดภัยเพียงพอสำหรับข้อมูลคุณภาพอากาศ (ไม่ใช่ข้อมูลการเงินหรือข้อมูลส่วนบุคคลที่ละเอียดอ่อน) มีการป้องกัน 2 ชั้น แอดมินทั่วไปเข้าตั้งค่าระบบไม่ได้
แยกได้ไหมว่าใครทำอะไร?
ได้ ทุกการกระทำบันทึกชื่อผู้ทำไว้ในระบบ (Activity Log) ดูย้อนหลังได้ว่าใครเพิ่ม/แก้/ลบอะไร เมื่อไหร่
ทำไมไม่แก้ระบบเดิม?
แก้มาแล้ว 3 รอบ ปัญหาอยู่ที่ตัว Supabase Auth เอง ซึ่งออกแบบมาสำหรับแอปขนาดใหญ่ (ผู้ใช้หลักพัน-หมื่น) ทีมเรามีแค่ 10-15 คน ใช้ระบบที่เหมาะสมกว่า
ถ้ามีคนใช้เยอะขึ้นในอนาคต?
ถ้าขยายเป็น 50+ คน สามารถเปลี่ยนกลับไปใช้ระบบ Auth ที่ซับซ้อนกว่าได้ เพราะโครงสร้างฐานข้อมูลไม่ได้เปลี่ยน เปลี่ยนแค่วิธีล็อคอิน
สวมรอยกันได้ไหม?
ไม่ได้ เพราะแต่ละคนมี PIN ส่วนตัว ถึงรู้ชื่อคนอื่น แต่ไม่รู้ PIN ก็เข้าไม่ได้
ข้อมูลเก่าหายไหม?
ไม่หาย ฐานข้อมูลเดิมทั้งหมด เปลี่ยนแค่วิธีล็อคอินเข้าหน้าแอดมิน ข้อมูลคุณภาพอากาศ, บ้าน, เซนเซอร์ ทุกอย่างยังอยู่ครบ
