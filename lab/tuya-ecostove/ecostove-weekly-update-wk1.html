⚠️ ข้อจำกัดที่พบ

ปัญหาในระบบเก็บข้อมูลปัจจุบัน

🔴 ปัญหาที่ 1 — Baseline ไม่น่าเชื่อถือ
ระบบใช้ 10 นาทีแรก ของ session เป็น baseline อัตโนมัติ แต่มีปัจจัยที่ควบคุมไม่ได้:
→ ฝั่งภาคสนาม: ไม่รู้ว่าชาวบ้านจุดเตาไปก่อนแล้วหรือยัง — ถ้าจุดก่อน baseline = ค่าควัน ไม่ใช่อากาศปกติ
→ ฝั่งระบบ: Cron ตรวจจับทุก 5 นาที → อาจพลาดจังหวะ + แม้เซนเซอร์ต่อ WiFi เสถียร ระบบก็ยังเก็บข้อมูลได้ไม่ครบทุก session
→ การหักลบ baseline เป็นสิ่งจำเป็นสำหรับงานวิจัย — ต้องมีวิธีที่น่าเชื่อถือ
🟡 ปัญหาที่ 2 — เซนเซอร์ออนไลน์ตลอด (บ้านมี WiFi)
บ้านที่มี WiFi → เซนเซอร์ต่อเน็ตตลอด → ระบบเริ่ม session อัตโนมัติ
→ ไม่มีคนควบคุม → ไม่รู้ว่าชาวบ้านใช้เตาจริงหรือเปล่า → ข้อมูลอาจปนกัน
→ เก็บได้แค่ 2 session/วัน แต่ไม่รู้ว่า session ไหนมีการใช้เตาจริง
🟡 ปัญหาที่ 3 — ไม่รู้จังหวะ "เริ่มใช้เตา" vs "อากาศปกติ"
ระบบปัจจุบันเก็บข้อมูลต่อเนื่อง โดยไม่แยกว่าช่วงไหน "ยังไม่ใช้เตา" ช่วงไหน "ใช้เตาแล้ว"
→ เทียบ eco vs old ได้ (approach D) แต่หักลบ baseline ไม่ได้อย่างแม่นยำ
→ ผลกระทบ: ค่าเฉลี่ยที่ได้อาจรวม ambient + cooking ปนกัน

ตัวอย่างปัญหา

Baseline อัตโนมัติ 10 นาที — มีจุดที่ต้องปรับปรุง

✅ สถานการณ์ที่ต้องการ

เซนเซอร์ต่อ WiFi → ระบบตรวจจับทันที → 10 นาที = อากาศปกติ ✅ → จุดเตา → เก็บค่าจริง
✅ Baseline = อากาศปกติจริงๆ → หักลบได้ถูกต้อง

⚠️ สถานการณ์ที่เกิดจริงได้

เซนเซอร์ต่อ WiFi → Cron ยังไม่ตรวจจับ (0–5 นาที) → จุดเตาระหว่างรอ → baseline = ค่าควัน
❌ Baseline ปนค่าควัน → หักลบแล้วผลคลาดเคลื่อน
🤖

ข้อจำกัดฝั่งระบบ

  • Cron ตรวจจับเซนเซอร์ทุก 5 นาที → อาจพลาดจังหวะ
  • แม้ WiFi เสถียร ระบบก็ยังเก็บข้อมูลได้ไม่ครบทุก session
  • ไม่สามารถรู้ได้ว่า "10 นาทีแรก" เป็นอากาศปกติจริงหรือไม่
🏠

ข้อจำกัดฝั่งภาคสนาม

  • ชาวบ้านอาจจุดเตาก่อนเซนเซอร์พร้อม
  • ไม่มีสัญญาณบอกระบบว่า "เริ่มใช้เตาแล้ว"
  • แต่ละบ้านมีพฤติกรรมต่างกัน ควบคุมยาก

🔧 ทางเลือก

แผนรับมือ — เลือกวิธีวัด Baseline

📌 แผนปัจจุบัน — เทียบ eco vs old ตรงๆ (ไม่หักลบ)
ระบบปัจจุบันเปรียบเทียบค่าเฉลี่ยระหว่าง eco กับ old โดยไม่หักลบ baseline

✅ ง่ายที่สุด ไม่ต้องเปลี่ยนอะไร
❌ ค่าที่ได้ไม่น่าเชื่อถือ เนื่องจากไม่มีการลบค่า Baseline ทิ้ง
✅ ทางเลือก A — อาสากดปุ่ม "เริ่มใช้เตา" (แนะนำ)
อาสาอยู่ที่บ้าน → เปิด Hotspot → ระบบเก็บ baseline → สังเกตชาวบ้านจุดเตา → กดปุ่ม 1 ครั้ง บนมือถือ → ระบบตัด baseline ตรงจุดนั้น

✅ ได้ baseline ตรงความเป็นจริง  |  ✅ อาสากดง่ายแค่ปุ่มเดียว  |  ✅ แก้ปัญหาเซนเซอร์ออนไลน์ตลอดด้วย
⚠️ ต้องมีอาสา 1 คน/บ้าน ตลอดการเก็บ
🔵 ทางเลือก B — แยก 2 รอบ (baseline แยก session)
รอบ 1: เปิดเซนเซอร์ก่อนใช้เตา 30 นาที → เก็บ baseline → ปิด
รอบ 2: จุดเตาแล้วค่อยเปิดใหม่ → เก็บค่าจริง

✅ แยกข้อมูลชัดเจน
⚠️ ยุ่งยาก ต้องเปิด-ปิด Hotspot 2 รอบ  |  ⚠️ ใช้ 4 session/วัน (baseline 2 + เก็บจริง 2) — เกินขีดจำกัดระบบ (สูงสุด 2/วัน) → ต้องปรับระบบ + เสี่ยง API เกิน Quota
🟡 ทางเลือก C — ตรวจจับจากข้อมูล (algorithm)
ระบบดูว่า PM2.5/CO2 พุ่งขึ้นตอนไหน → ใช้ช่วงก่อนพุ่งเป็น baseline

✅ อัตโนมัติ 100% ไม่ต้องกด
⚠️ ไม่แม่นเสมอ (ถ้าค่าค่อยๆขึ้น ไม่ใช่ spike)  |  ⚠️ ต้องเขียน algorithm ซับซ้อน  |  ⚠️ ยากอธิบายใน paper
