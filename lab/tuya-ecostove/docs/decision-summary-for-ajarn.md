# EcoStove: สรุปสถานการณ์และทางเลือก

> เอกสารสำหรับประกอบการตัดสินใจ
> วันที่: 2026-01-26

---

## สถานการณ์ปัจจุบัน

### Sensor ที่เลือกใช้
- **รุ่น:** Tuya MT15/MT29 Air Quality Sensor
- **ราคา:** ~1,500 บาท/เครื่อง
- **วัดได้:** PM2.5, PM10, PM1.0, CO2, Temperature, Humidity

### ข้อจำกัดที่พบ

```
❌ Sensor ไม่มี memory เก็บข้อมูล
   → ต้องดึงข้อมูลแบบ real-time เท่านั้น
   → ถ้าไม่มี WiFi ขณะวัด = ข้อมูลหาย

❌ Tuya Cloud API มีข้อจำกัด (Trial Edition)
   → 26,000 API calls/เดือน
   → ห้ามใช้เชิงพาณิชย์/วิจัย
   → เก็บได้สูงสุด ~12 บ้าน (ถ้าเก็บ 2 ครั้ง/วัน, 3 ชม./ครั้ง)
```

---

## คำถามที่ต้องการคำตอบ

| # | คำถาม | ผลกระทบต่อการตัดสินใจ |
|---|-------|----------------------|
| 1 | งบประมาณโปรเจคมีเท่าไหร่? | กำหนดว่าใช้วิธีไหนได้ |
| 2 | บ้านกลุ่มตัวอย่างกี่บ้านมี WiFi? | ถ้าไม่มี ต้องลงทุนเพิ่ม |
| 3 | ต้องการกี่บ้านขั้นต่ำ? | ถ้าลดได้ = ลดค่าใช้จ่าย |
| 4 | ยอมรับข้อมูลไม่ต่อเนื่องบางบ้านได้ไหม? | ถ้าได้ = ใช้ Hybrid |
| 5 | มีทีม IT ดูแลระบบไหม? | บางวิธีต้องมีคนดูแล |

## ทางเลือก

### ทางเลือก A: ใช้เฉพาะบ้านที่มี WiFi (Cloud API)

| รายการ | จำนวน | ราคา |
|--------|-------|------|
| Sensor MT15/MT29 | 12 เครื่อง | 18,000 |
| **รวม** | | **18,000** |

**เงื่อนไข:**
- ใช้ได้เฉพาะบ้านที่มี WiFi
- จำกัด ~12 บ้าน (เพราะ API limit)
- เสี่ยงผิด TOS ของ Tuya (Trial ห้ามใช้วิจัย)
- ได้ sample น้อย อาจมี bias

---

### ทางเลือก B: ติด 4G Router ทุกบ้าน

| รายการ | จำนวน | ราคา |
|--------|-------|------|
| Sensor MT15/MT29 | 30 เครื่อง | 45,000 |
| 4G Router | 30 เครื่อง | 60,000 |
| ค่าเน็ต 6 เดือน | 30 × 250 × 6 | 45,000 |
| **รวม** | | **150,000** |

**เงื่อนไข:**
- ไม่ต้องมี WiFi เดิม (ใช้ 4G)
- ได้ครบ 30 บ้าน
- ง่ายที่สุด ไม่ต้องมีคน IT
- แพงที่สุด

---

### ทางเลือก C: Hybrid (WiFi + จดมือ)

| รายการ | จำนวน | ราคา |
|--------|-------|------|
| Sensor MT15/MT29 | 30 เครื่อง | 45,000 |
| **รวม** | | **45,000** |

**เงื่อนไข:**
- บ้านมี WiFi → ใช้ Cloud API
- บ้านไม่มี WiFi → อาสาจดมือ
- ข้อมูลคุณภาพไม่เท่ากัน
- ต้องใช้แรงงานอาสา

---

### ทางเลือก D: เปลี่ยน Sensor

**เงื่อนไข:**
- ใช้ Sensor ที่มี SD Card หรือ memory ในตัว
- แก้ปัญหาได้ตรงจุด
- ต้อง research หารุ่นที่เหมาะ
- ราคายังไม่ทราบ

---

### ทางเลือก E: Raspberry Pi เป็น Local Logger ⭐ แนะนำ

| รายการ | จำนวน | ราคา |
|--------|-------|------|
| Sensor MT15/MT29 | 30 เครื่อง | 45,000 |
| Raspberry Pi Kit | 30 ชุด | 30,000 |
| **รวม** | | **75,000** |

**เงื่อนไข:**
- ใช้ได้เฉพาะบ้านที่มี WiFi
- ได้ครบ 30 บ้าน
- ไม่มี API limit, ไม่เสี่ยง TOS
- ต้องมีคน IT setup และดูแล
- รองรับ workflow ที่ต้องการ (เก็บ → อาสามาดึง)

## ตารางเปรียบเทียบ

| ทางเลือก | งบ (30 บ้าน) | จำนวนบ้าน | ต้อง WiFi | ต้อง IT | ความเสี่ยง |
|----------|-------------|-----------|----------|---------|-----------|
| **A: Cloud API** | 18,000 | ~12 | ✅ | ❌ | สูง |
| **B: 4G Router** | 150,000 | 30 | ❌ | ❌ | ต่ำ |
| **C: Hybrid** | 45,000 | 30 | บางบ้าน | ❌ | ปานกลาง |
| **D: เปลี่ยน Sensor** | ? | 30 | ❌ | ❌ | ต่ำ |
| **E: Pi** ⭐ | 75,000 | 30 | ✅ | ✅ | ต่ำ |

---

## คำแนะนำ

```
งบ < 20,000        → ทางเลือก A (แต่ได้แค่ 12 บ้าน + เสี่ยง TOS)
งบ 45,000-50,000   → ทางเลือก C (Hybrid)
งบ 75,000-80,000   → ทางเลือก E ⭐ (แนะนำ ถ้ามี IT)
งบ > 100,000       → ทางเลือก B (ง่ายที่สุด)
```
