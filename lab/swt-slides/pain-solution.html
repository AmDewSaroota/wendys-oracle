Mobile Web App: Pain Points

Traveler Experience Gaps
🎫

การเดินทางไม่ได้จบที่การจองตั๋ว

ระบบจองตั๋วปัจจุบันเน้น "การทำธุรกรรม" — ซื้อตั๋วเสร็จก็จบ แต่ผู้โดยสารยังต้องการข้อมูลระหว่างทาง ความช่วยเหลือ และการวางแผนปลายทาง

📝

ขั้นตอนการจองซับซ้อนเกินไป

ต้องกรอกข้อมูลผู้เดินทางทุกคนก่อน จึงจะทราบว่าที่นั่งว่าง — หากเต็มต้องเริ่มใหม่ทั้งหมด

📌 ข้อมูลจากผู้ใช้จริง (Pantip, App Store)
🔀

เส้นทางเชื่อมต่อต้องหาเอง

หากไม่มีรถไฟตรง ผู้โดยสารต้องค้นหาจุดเปลี่ยนขบวนเองทีละสาย ไม่มีระบบแนะนำเส้นทางเชื่อมต่อผ่านสถานี hub

⏱️

ข้อมูลสถานะขบวนรถไม่ทันการณ์

ผู้โดยสารพบว่ารถไฟล่าช้าหลายชั่วโมง แต่แอปยังแสดงเวลาตามตารางเดิม — วางแผนต่อไม่ได้

📌 ข้อร้องเรียนที่พบบ่อยที่สุด (App Store Reviews)
🗺️

ข้อมูลท่องเที่ยวแยกจากระบบจอง

ไม่มีการแนะนำกิจกรรม ทัวร์ หรือร้านค้ารอบสถานีปลายทาง — ต้องออกจากแอปไปหาจากแหล่งอื่น

📊

ผู้บริหารขาด Data-Driven Insights

ไม่มี Dashboard แสดงข้อมูล Conversion Funnel, พฤติกรรมผู้ใช้ หรือความพึงพอใจ เพื่อใช้ตัดสินใจเชิงนโยบาย

การยกระดับประสบการณ์ผู้โดยสาร

Enhancing the Passenger Experience
ความต้องการผู้โดยสาร สถานการณ์ปัจจุบัน แพลตฟอร์ม SWT ✦
🔀 ค้นหาเส้นทาง เลือกได้เฉพาะเส้นทางตรง Smart Routing + เชื่อมต่อผ่าน hub อัตโนมัติ
⚡ ความเร็วในการแสดงผล โหลดช้าในช่วงผู้ใช้หนาแน่น Edge Computing — แสดงผลเร็วสม่ำเสมอ
💺 สถานะที่นั่ง กรอกข้อมูลผู้เดินทางก่อนถึงจะรู้ว่าว่าง Availability-First — รู้ทันทีตั้งแต่ค้นหา
⏱️ สถานะขบวนรถ อ้างอิงตามตารางเวลาหลัก อ้างอิงตำแหน่งจริง (Real-time Tracking)
📌 ข้อมูลท่องเที่ยว ต้องหาข้อมูลจากแหล่งอื่น Tour Packages จองได้ในแอปเดียว
📱 ช่วยเหลือระหว่างทาง โทรสายด่วน 1690 AI Chatbot (RailBot) ตอบทันที TH/EN
🏆 แรงจูงใจกลับมาใช้ ไม่มีระบบ Loyalty Gold → Platinum + แต้มสะสม + คูปอง
🚉 นำทางในสถานี ไม่มีแผนที่สถานี Interactive Map + คำนวณระยะทาง/เวลาเดิน
📊 ข้อมูลสำหรับผู้บริหาร ไม่มี Analytics Dashboard Stats Dashboard — Booking / Revenue / Heatmap / Satisfaction
