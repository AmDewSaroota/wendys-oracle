# Lesson: Report Communication Priorities

**Date**: 2026-04-02
**Source**: PV28 vs MT13W report session
**Confidence**: High (confirmed by DewS)

## Pattern

When preparing technical reports for advisors/stakeholders:

1. **Lead with the domain expert's priority** — DewS said PM2.5 is the main problem, TVOC is bonus. Don't let technical excitement (auto cloud sync!) override what the research actually needs.

2. **Don't oversell unverified capabilities** — TVOC cloud pulling works in API but hasn't been tested through the full system pipeline. Say "ยังไม่ได้ทดสอบดึงจริงผ่านระบบ" not "ทำได้แน่นอน!"

3. **Calibrate severity language** — A fixable issue (3 button presses) shouldn't use red/alarming visuals. Use orange/minor. Reserve red for actual blockers.

4. **Mobile-first for LINE-shared reports** — Advisors open links on phones. Default to 1-column layout, expand on desktop.

5. **Targeted edits over replace_all** — Blanket string replacement can create duplicate text in rows with similar patterns.
