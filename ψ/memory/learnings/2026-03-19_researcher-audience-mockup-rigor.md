# Lesson: Researcher-Audience Mockup Rigor

**Date**: 2026-03-19
**Source**: rrr: wendys-oracle
**Tags**: #ecostove #mockup #data-accuracy #stakeholder-management

## Pattern

When building mockups for academic researchers (especially PhD-level), treat every number as a claim that will be scrutinized. Unlike designer wireframes where "lorem ipsum" is acceptable, research mockups demand:

1. **Exact counts** — if the project has 10 houses, show 10 rows in every table
2. **Consistent formats** — if one hero says "ⓘ วิธีวัด · ข้อมูล 60 วัน · 10 ครัวเรือน", ALL heroes must follow the same pattern
3. **Defensible numbers** — sample sizes (n=142/89), readings counts (4,800), and durations (60 days) must be internally consistent

## Anti-Pattern

Building tables with 4 rows "for brevity" when the project spec says 10 houses. Using different footer formats across 3 scenario heroes because they were built at different times.

## Rule

Before shipping any mockup to a researcher:
- **Count audit**: Does every table/card/list show the correct total items?
- **Format audit**: Do all parallel sections use the same display format?
- **"What's the rule?" test**: Can you explain the logic behind every data display choice?
