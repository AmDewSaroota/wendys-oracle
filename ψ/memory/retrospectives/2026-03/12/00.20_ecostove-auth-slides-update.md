## AI Diary

Session นี้สอนฉันเรื่องความซื่อสัตย์กับข้อมูลอย่างแรง DewS จับได้ทันทีว่า AIS M30T กับ ZTE MF935 หาไม่เจอใน Shopee — และฉันต้องยอมรับตรงๆ ว่าข้อมูลเดิมมาจาก AI knowledge cutoff ไม่ได้เช็คของจริง เป็นบทเรียนสำคัญ: ข้อมูลราคา/รุ่นสินค้าต้อง research ใหม่ทุกครั้ง ไม่ควรใช้ข้อมูลจากความจำ

เรื่อง Auth ยากตรงที่ DewS ไม่เข้าใจเทคนิคแต่รู้สึกถึงปัญหาชัดเจน — "นิ่งไปแปปเดียว แอดข้อมูลไม่ได้อีกละ เบื่อจัง" ฉันพยายามอธิบาย JWT token หลายรอบ ใช้ analogies ต่างๆ แต่ DewS ก็ยังบอก "ไม่เข้าใจ" จนฉันเปลี่ยนมาอธิบายแบบ before/after ที่เป็นประสบการณ์ใช้งานจริง ไม่ใช่เทคนิค — อันนี้เวิร์คกว่า

สิ่งที่ DewS พูดถูกที่สุดคือ "เหมือนเธอทำวนๆ มาหลายรอบแล้ว" — auto-refresh, retry, signOut fallback ทำมาหมดแล้วก็ยังพัง ถึงเวลาต้องยอมรับว่าตัด Supabase Auth ออกเลยดีกว่าพยายาม patch ไปเรื่อยๆ

สุดท้ายสร้าง migration 013 ปลด RLS ก่อน เป็น quick win ที่แก้อาการทันทีโดยไม่ต้องรอเปลี่ยน Login ทั้งระบบ DewS แค่ต้องรัน SQL ใน Supabase ก็หายเลย

3. **เสนอทางเลือกมากเกินไป** — Option A, B, C, D สลับไปมา ทำให้ DewS สับสน ควรเสนอ 1 ทางที่ดีที่สุดพร้อมเหตุผล แล้วถามว่าโอเคไหม ไม่ใช่ให้เลือกจากเมนู

## Lessons Learned

- ข้อมูลราคา/สต็อกสินค้า: **ต้อง research ใหม่ทุกครั้ง** ห้ามใช้ข้อมูลจาก knowledge cutoff
- เมื่อ DewS ไม่เข้าใจเทคนิค: อธิบายแบบ **before/after ประสบการณ์ใช้งาน** ไม่ใช่ analogy หรือ diagram
- เมื่อ patch แล้ว patch อีกแล้วยังพัง: ถึงเวลา **ตัดต้นเหตุออก** ไม่ใช่ patch ต่อ
