## Lessons Learned

- **Design for production scale, not mock data** — อย่าให้ test data (10 บ้าน, 60 วัน) shape design ของ production system (10 บ้าน, 6 เดือน)
- **Know your own system** — ถ้าสร้าง ambient subtraction แล้ว ต้องจำว่า net values เทียบข้ามช่วงเวลาได้ อย่า reasoning ผิดเพราะลืม feature ตัวเอง
- **Pure function extraction = testability gold** — แยก logic ออกจาก I/O → test 94 กรณีใน 84ms ไม่ต้อง mock
- **Timezone tests need manual day-of-week verification** — อย่า assume ว่า UTC date X = Thai day Y โดยไม่คำนวณ
