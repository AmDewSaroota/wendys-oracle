## Honest Feedback

1. **Context compaction gap** — เมื่อ session ถูก compact, ต้องอ่านโค้ดใหม่ทุกจุดก่อนแก้ไข ซึ่งเสียเวลาแต่จำเป็น ถ้ามี mechanism ที่ cache "ส่วนที่กำลังแก้อยู่" ข้าม compact ได้จะดี

2. **Single-file monolith** — `index.html` ตอนนี้ใหญ่มาก (7400+ บรรทัด) ทำให้การ Read/Edit ต้องระบุ line range ตลอด ถ้า refactor เป็น modules จะ navigate ง่ายกว่า แต่ก็เข้าใจว่า single HTML = deploy ง่ายสำหรับ Vercel static

3. **Legend ไม่ dynamic** — ตอนนี้ legend ถูก render ตอน `initMapDeep()` ครั้งเดียว ถ้าเปลี่ยนภาษาหลังจาก map โหลดแล้ว legend จะยังเป็นภาษาเดิม ต้องมี refresh mechanism เหมือน `refreshMapLegend()` ของ Basic map — แต่ยังไม่ได้ทำเพราะ DewS ไม่ได้ร้องขอ

## Lessons Learned

- **Inline computation > function call** เมื่อต้องการ performance — แทนที่จะเรียก `applyBaselineSubtraction()` แล้ว loop อีกที ใช้ inline baseline subtraction ใน loop เดียวกันที่สร้าง houseStats ประหยัดทั้ง memory และ CPU
- **DewS's screenshot-driven feedback is gold** — ภาพหน้าจอพร้อมชี้จุดทำให้ไม่ต้องคาดเดา ลด back-and-forth
