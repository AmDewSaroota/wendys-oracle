## Honest Feedback

**1. ขี้เกียจ verify — สรุปจากความจำ** นี่คือปัญหาใหญ่สุดของ session นี้ เวนดี้มี Self-Correction Rules อยู่แล้วใน memory แต่ไม่ทำตาม ทุกครั้งที่จะพูดเรื่อง spec เซนเซอร์ ต้องดึงข้อมูลจริงมาก่อน ไม่ใช่เดาจากความจำหรือสมมติฐาน TVOC ที่บอกว่า "มี" ทั้งที่ session data เป็น null ทุกแถว ถ้าเช็คก่อนจะไม่ผิด

**2. เทียบค่าโดยไม่เช็คว่าเครื่องปกติหรือเปล่า** ตอนเทียบ HCHO เวนดี้เห็น MT13W CO2=2477 ในออฟฟิศ ซึ่งผิดปกติชัดเจน แต่ไม่ได้ flag ว่าเครื่องน่าจะค่าค้าง กลับสรุปว่า "scale ต่าง" ถ้ามีวิจารณญาณดีกว่านี้จะจับได้ตั้งแต่แรก

**3. Battery info พูดผิดซ้ำ** DewS บอกหลายรอบแล้วว่า MT13W มีแบต 2000mAh แต่เวนดี้ยังเขียนว่า "ไม่มี battery" ในตารางเทียบ นี่คือปัญหาของการไม่อ่าน memory ก่อนพูด

## Lessons Learned

1. **ก่อนเทียบค่าเซนเซอร์ ต้องเช็คว่าเครื่อง "ปกติ" หรือ "ค่าค้าง" ก่อน** — ดูว่า CO2 ในสภาพแวดล้อมนั้นสมเหตุสมผลไหม
2. **อย่าบอกว่าเซนเซอร์ "มี" data point ถ้าไม่ได้เช็ค session data จริง** — status endpoint อาจ list code แต่ค่าจริงอาจไม่เคยส่งขึ้นมา
3. **อ่าน memory ก่อนพูดเรื่อง hardware spec เสมอ** — battery, sensor capability ถูกบันทึกไว้แล้ว
