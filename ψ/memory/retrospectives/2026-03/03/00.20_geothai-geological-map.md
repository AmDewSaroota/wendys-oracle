## Key Architecture Decisions

1. **Single HTML file** — ไม่ต้อง build, เปิดจาก browser ได้เลย
2. **Leaflet.js + MarkerCluster** จาก CDN — lightweight, mobile-friendly
3. **`window.GEOTHAI_DATA` namespace** — แยก data files ออกจาก logic, โหลดผ่าน `<script src>`
4. **CartoDB Dark Matter** tiles — dark theme ที่เข้ากับ geology data
5. **i18n via `data-i18n` attributes** — สลับ TH/EN ได้ทันที ทั้ง UI + popup content
6. **Mobile: bottom tabs + bottom sheet + full-screen search overlay** — หลัง 2 รอบ UX fixes

## Honest Feedback

**Friction Point 1: Mobile-first ที่ไม่ได้ test จริงบนมือถือ**
ฉันเขียน CSS responsive ทั้งหมดบน desktop แล้วคาดหวังว่ามันจะทำงานได้บนมือถือ ซึ่งมันไม่ได้ Legend ที่ดูเล็กบน desktop กลับบังทั้งจอบนมือถือ Search dropdown ที่อยู่ล่างจอก็ตกขอบ ต้องแก้ 2 รอบ ควรทำ mobile-first จริงๆ ตั้งแต่แรก หรือใช้ Chrome DevTools responsive mode ก่อน deploy

**Friction Point 2: Data accuracy ยากจะ verify**
ข้อมูล 55 แหล่งแร่ + 16 รอยเลื่อนถูกสร้างจาก research แต่พิกัดบางจุดอาจคลาดเคลื่อน ฉันไม่สามารถ verify ทุกจุดได้จริงว่าตรงกับข้อมูล DMR ต้นฉบับ ซึ่งเป็น limitation ที่ต้อง acknowledge

**Friction Point 3: Context window pressure จาก direction change**
การเปลี่ยนทิศกลางเซสชันทำให้ context window มี "ขยะ" จากเกม capybara ค้างอยู่ — game code, research เทรนด์เกมเด็ก, Roblox data — ไม่ได้ใช้เลยแต่กินพื้นที่ context ถ้ามี way to "flush" context ที่ไม่เกี่ยวข้องได้จะดี

## Lessons Learned

1. **Mobile UX ต้อง test บนมือถือจริง** — CSS media queries + Chrome DevTools ไม่เพียงพอ ต้องมีรอบ real device testing ก่อน deploy
2. **Parallel agents ลดเวลาได้มาก** — 3 data files + 1 main file พร้อมกัน เสร็จใน ~20 min แทนที่จะ ~60 min sequential
3. **Full-screen overlay > bottom sheet สำหรับ search** — Bottom sheet ที่มี dropdown results จะมีปัญหาขอบจอเสมอ full-screen overlay ตัดปัญหานี้ได้หมด
