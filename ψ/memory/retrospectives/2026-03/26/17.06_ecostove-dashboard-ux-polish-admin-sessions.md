3. **Hero card "-- %" fix** — `detectScenario()` didn't handle "eco data only" case. Added 'no-old' scenario with muted stone/amber hero card.
4. **Stat cards redundancy** — Same long explanation repeated 3x on cards. Replaced with short label per card + shared banner note.
5. **Drill-down removal** — Removed per-house drill-down charts from Deep Insights (but kept Session Timeline table) per DewS's clarification.
6. **Admin Sessions tab** — Built `loadAdminSessions()` with date-grouped cards, filters, device dropdown auto-population.
7. **Sessions -> Data sub-tab** — DewS wanted it nested inside "ข้อมูล" as 3rd sub-tab (เซนเซอร์ | กรอกด้วยมือ | เซสชันย้อนหลัง).

## Honest Feedback

**1. Deploy cycle ถี่เกินไป** — Deploy ไป 4-5 ครั้งใน session เดียวเพราะ DewS ทดสอบบนโปรดักชัน ถ้ามี local preview (localhost) จะลดรอบได้มาก แต่ด้วย monolithic HTML + Supabase connection ทำให้ local preview ไม่ practical ตอนนี้

**2. Monolith index.html ยากขึ้นเรื่อยๆ** — ไฟล์เดียว 10,000+ บรรทัด ทุกครั้งที่แก้ต้อง grep หา function, line number, context แล้ว edit ทีละจุด ถ้า session ยาวขึ้นอีก context window จะเป็นปัญหา

**3. Context recovery หลัง summarization** — Session นี้เริ่มจาก summarized context ต้องใช้เวลา re-orient ว่าทำอะไรค้างไว้ตรงไหน โชคดีที่ handoff file + summary ครบ แต่ถ้าไม่มี handoff จะยากกว่านี้มาก

## Lessons Learned

- **ฟังให้จบก่อนลงมือ** — "ลบ session" อาจหมายถึง "ลบ drill-down แต่เก็บ timeline" ต้อง clarify ก่อน action
- **Sub-tab > Main tab** สำหรับ feature ที่ใช้ไม่บ่อย — ลดความรกของ navigation
- **Dynamic pointRadius** แก้ปัญหา sparse data ใน Chart.js ได้ดี — pattern ที่ reuse ได้
