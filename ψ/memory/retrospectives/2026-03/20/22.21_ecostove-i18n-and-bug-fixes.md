## Lessons Learned

1. **Data cutoff ต้อง design ตาม use case ไม่ใช่ hardcode** — 60 วันอาจพอสำหรับ monitoring ปกติ แต่ research project ที่มี phase 1 + phase 2 ต้องเห็นข้อมูลทั้ง 2 ช่วง 180 วันเป็น compromise ที่ดีระหว่าง coverage กับ performance
2. **data-i18n-html สำหรับ rich content** — ถ้า element มี HTML (bold, spans) ใน content ที่ต้องแปล ใช้ data-i18n (textContent) ไม่ได้ ต้องแยก handler สำหรับ innerHTML
3. **Performance budget สำคัญ** — 65K records = 30+ วินาที load time ไม่ acceptable สำหรับ dashboard ต้อง pre-aggregate (daily_summaries) หรือ paginate UI
