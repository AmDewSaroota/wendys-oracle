## Honest Feedback

**Friction 1**: The conversation was extremely long and covered too many topics in one session. From Deep Insights filter redesign to mock data reset to admin tab polish — these should ideally be separate sessions. The context compaction happened because we packed too much into one conversation, and some nuance was lost in the summary.

**Friction 2**: The i18n system is getting unwieldy. Every UI text change requires updating both the HTML default text AND the i18n dictionary, and they can get out of sync easily. The old tab names ("อัตโนมัติ", "TVOC") were still in the i18n dictionary while the HTML already showed the new names. This caused confusion about whether the rename was actually done.

**Friction 3**: The single-file architecture of index.html (now 7000+ lines) makes every edit a needle-in-haystack exercise. Reading 80 lines at a time to find the right function, then making surgical edits — it works, but it's fragile. A single misplaced closing tag could break everything, and there's no build step to catch errors.

## Lessons Learned

- When DewS asks a question mid-task, answer it directly with evidence (show the code line), then resume — don't lose the thread
- Mock data strategy: clearing everything and regenerating is cleaner than patching messy real data
- CO comes from 2 sources (sensor auto + manual) — both should be visible in their respective tables
- Export functions should always offer the same UX pattern (date range modal + Excel/CSV)
