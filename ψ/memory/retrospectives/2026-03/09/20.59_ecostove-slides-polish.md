## Honest Feedback

**1. Slide numbering — complexity overhead เกินจำเป็น**
การ rename slide numbers ทุกครั้งที่ delete/add/move slide เป็น error-prone operation มาก session นี้ต้อง rename หลายรอบ (15→13→12→9) แต่ละรอบต้องระวัง collision ของตัวเลขซ้ำ ควรมี script ช่วย renumber อัตโนมัติ หรือ JS แบบ dynamic count ที่ไม่ต้อง hardcode เลยสักตัว

**2. "ซ่อน" กับ "ลบ" ไม่ได้แยกกันชัดเจน**
DewS พูดว่า "ซ่อนไว้" แต่ในบริบทของ HTML presentation ฉันตัดสินใจ delete ออกเลย เพราะ HTML comment + JS `querySelectorAll` จะยุ่งยาก แต่ DewS อาจหมายความว่าเก็บไว้ใน comment จริงๆ ควรถามให้ชัดก่อน — "ซ่อน = comment out ยังอยู่ในไฟล์ หรือ ลบออกจริงๆ?"

**3. Slide content design — ทำได้ดีขึ้นได้อีก**
ตอนที่ merge slides ฉัน "ย่อ" เนื้อหาเอง โดยไม่ได้ถาม DewS ว่าอยากเก็บอะไรไว้บ้าง เช่น slide 3 เดิมมี 3 issue boxes ยาวมาก ฉันย่อเหลือ 1 บรรทัด/อัน ซึ่งโอเค แต่ควรบอก DewS ว่า "ฉันย่อเนื้อหานี้ไป ถ้าอยากดูรายละเอียดเดิมบอกได้"

## Lessons Learned

1. **Slide number renaming: specific first, replace_all last** — เมื่อต้องเปลี่ยน "N / total" ของแต่ละ slide ต้องเรียงจากเลขที่ unique ที่สุดก่อน (เช่น largest number) แล้วค่อย replace_all ทีหลัง เพื่อหลีกเลี่ยง collision
2. **"ซ่อน" vs "ลบ" ใน presentation context** — ถามให้ชัดว่าอยาก comment out หรือ delete จริงๆ
3. **ย่อ content = ต้องแจ้ง** — เมื่อ merge slides แล้วต้อง trim content ควรบอกผู้ใช้ว่า trim อะไรออกไปบ้าง
