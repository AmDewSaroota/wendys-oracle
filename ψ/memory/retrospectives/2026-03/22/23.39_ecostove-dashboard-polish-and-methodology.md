## Honest Feedback

**Friction 1: Chart debugging was too circular.** I spent too many rounds theorizing about why band and mean didn't match without asking DewS to hover earlier. Should have asked for tooltip verification in the first response instead of the fourth. The "verify before theorizing" principle should be my default, not my fallback.

**Friction 2: Overconfident about cache being the issue.** I suggested browser cache as the reason the fix didn't work, but DewS tested in Guest mode and it was still the same. I should have checked whether `applyBaselineSubtraction` actually affects mock data (it doesn't — no session_id) before suggesting the fix was deployed correctly.

**Friction 3: Multiple deploys for iterative fixes.** Three deploys in one session because each fix was incremental. Should have batched the visual changes (opacity, thickness) with the data source fix in one deploy instead of deploying, finding it still looks the same, then deploying again.

## Lessons Learned

- Mock data without sessions bypasses `applyBaselineSubtraction` completely — netData = rawData. Chart fixes targeting baseline subtraction have no visible effect on mock data.
- When debugging visual chart issues: ask user to hover/tooltip FIRST, then theorize. User's eyes + tooltip = fastest verification path.
- Chart.js `fill: '+1'` at 8% opacity on white background is effectively invisible for wide bands. 15%+ needed for visible bands.
