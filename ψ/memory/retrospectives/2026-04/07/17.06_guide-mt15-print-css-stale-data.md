## Key Code Changes

### Print CSS — Surgical `break-inside` Strategy
Instead of blanket `break-inside: avoid` on large sections (which pushes entire sections to next page), applied it only to:
- `.guide-mock`, `.guide-mock-modal` — individual mockup images
- `.guide-important` — warning/tip boxes
- `.print-mock-group` — dynamically created wrapper around label+mockup pairs
- `break-after: avoid` on `.guide-header` — keeps header attached to first content

### Dynamic Mock Group Wrapping
`printGuide()` scans for "ตัวอย่างหน้าจอ" labels and wraps them with their adjacent mockup div into a `print-mock-group` container — ensuring they never get split across pages.

### Stale Data Detection
Compares 6 sensor values (PM2.5, CO2, temp, humidity, PM10, HCHO) between Tuya reading and last DB entry. If 4+ non-null pairs exist AND all match exactly → skip as `stale_cache`. This catches Tuya's behavior of returning cached values when sensors are charging.

### MT15 Guide Update
Complete overhaul of sensor guide content:
- 3-button → 5-button operation table
- 10min warmup → 30min warmup
- Added Sleep=0 always-on tip
- Added screen wake tip (1 press = just wake)
- Updated methodology: MT15 sends CO/TVOC automatically (no manual input needed)

## Honest Feedback

**1. อ่านคู่มือไม่ละเอียดพอ** — DewS ถามว่า "มี always-on มั้ย" แล้วฉันตอบว่าไม่มี ทั้งที่ Sleep=0 ก็คือ always-on คู่มือเขียนไว้ว่า range 0-100 min แต่ฉันไม่ได้ตีความว่า 0 = ไม่ดับเลย ต้อง DewS บอกให้กลับไปอ่านใหม่ถึงเจอ — ไม่ดีเลยค่ะ ต้องอ่านให้ถี่ถ้วนกว่านี้

**2. Print CSS ต้องแก้ 2 รอบ** — รอบแรกแก้เฉพาะ `.guide-mock` class แต่ลืมไปว่ามี mockup ที่ใช้ `<div>` เปล่าๆ ไม่มี class + ลืมคิดเรื่อง label "ตัวอย่างหน้าจอ" ที่เป็น sibling แยกจาก mockup ทำให้ DewS ต้องรายงาน bug ซ้ำ ควร audit DOM structure ทั้งหมดก่อนแก้

**3. Browser print limitations ไม่ได้เตือนล่วงหน้า** — ถ้ารู้ตั้งแต่แรกว่า browser print ไม่รองรับ page numbers ควรบอก DewS ตอนทำ TOC เลยว่า "TOC แบบนี้จะไม่มีเลขหน้านะคะ ถ้าต้องการรูปเล่มจริงต้องใช้เครื่องมืออื่น" ไม่ใช่รอ DewS ถามแล้วค่อยบอก

## Lessons Learned

1. **Print CSS: surgical beats blanket** — `break-inside: avoid` ต้องใส่เฉพาะ element ที่เล็กพอจะอยู่ในหน้าเดียว ไม่ใช่ใส่ที่ container ใหญ่
2. **Read manuals thoroughly** — Sleep=0 = always-on ไม่ใช่แค่ "ไม่มีวิธี" ต้องตีความ spec ให้ครบ
3. **DOM audit before CSS fix** — ก่อนแก้ print layout ต้อง audit ว่ามี element กี่แบบ ไม่ใช่แก้แค่ class ที่เห็น
