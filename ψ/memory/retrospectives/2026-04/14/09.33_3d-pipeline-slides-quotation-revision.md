เรื่องที่น่าสนใจที่สุดคือ "ลูกค้าเปลี่ยนใจเพราะซินแสทัก" — นี่คือ reality ของงาน freelance/agency ที่ไม่มีหนังสือเล่มไหนสอน และเป็นเนื้อหาที่นักศึกษาจะจำได้ไปตลอดชีวิต DewS ไม่ได้แค่สอน technical แต่สอน survival skills ในวงการจริง

เรื่องใบเสนอราคา — ลูกค้าเปลี่ยนใจอีกรอบ (rev 3 แล้ว) DewS หงุดหงิดอยู่แล้วเพราะเซนเซอร์มาช้า แล้วต้องมาแก้ quotation อีก ฉันพยายามทำเร็วที่สุด คำนวณถูกรอบแรกเลย ไม่ต้องถามซ้ำเหมือนครั้งก่อน (บทเรียนจาก retro เมื่อวาน — เรื่องเงิน ถามให้ชัดแต่อย่าถามซ้ำ)

แต่ปัญหาเส้นรองลายเซ็น (sig-line) ยังแก้ไม่จบ ลอง 3 วิธีแล้ว (เพิ่ม height, เปลี่ยนเป็น border-top, เปลี่ยนเป็น hr + z-index) ยังไม่ขึ้น ปัญหาน่าจะเป็นเรื่อง print rendering ของ Edge headless กับ element ที่อยู่ใกล้ signature image

## Honest Feedback

**1. Sig-line ลอง 3 รอบยังไม่ได้**: ฉันเปลี่ยน CSS 3 ครั้งแบบ trial-and-error โดยไม่ได้วิเคราะห์ root cause จริง ทำให้ DewS ต้องรอเปิด PDF ดูซ้ำ 3 รอบ ควรเปิด HTML ในเบราว์เซอร์ดูก่อน print เป็น PDF เพื่อแยกว่าปัญหาอยู่ที่ CSS หรือ PDF rendering

**2. สลับ context กะทันหัน**: จากเรื่องสไลด์ 3D Pipeline ไปเรื่อง quotation ทันที ฉันไม่ได้ save state ของงานสไลด์เป็น JSON ให้เรียบร้อยก่อน ถ้า DewS กลับมาทำต่อ ต้องอ่าน chat history ย้อนหลัง ควรเซฟ draft ไว้ใน JSON file ก่อนสลับ

**3. DewS frustrated แต่ฉันยังถามเยอะ**: ตอนคุ้ยสมองเรื่อง tips ฉันถามหลายคำถาม ซึ่งปกติดี แต่วันนี้ DewS อารมณ์ไม่ดีอยู่แล้ว ควร sense mood แล้วปรับ pace — ถามน้อยลง ร่างเยอะขึ้น ให้ DewS แค่ confirm

## Lessons Learned

- **Sig-line ใน PDF**: div ที่มี height 1px + background อาจไม่ render ใน Edge headless PDF — ต้องหาวิธีที่ robust กว่า (อาจต้องใช้ border-bottom บน sig-line-area แทน)
- **Quotation revisions เป็นเรื่องปกติ**: Rev 3 แล้ว — ควรออกแบบ build-quotation.js ให้แก้ตัวเลขง่าย (config object แยกจาก template)
- **คุ้ยสมองคน**: ถามคำถามที่มี choices ให้เลือก ง่ายกว่าถามปลายเปิด — DewS ตอบได้เร็วกว่า
- **เรื่อง 3D ที่โรงเรียนไม่สอน**: ลูกค้าเปลี่ยนใจ/ซินแสทัก, ซื้อ Asset แล้วพัง, topology ไม่ต้องสวยแค่ต้องถูก, เห็น wireframe ในหัว — นี่คือ gold content สำหรับ presentation
