## AI Diary

This session was a masterclass in pixel-perfect iteration with a designer's eye. DewS sees differences that I initially dismissed as minor — the font rendering difference between Sarabun and Tahoma, a few points of font size, the position of a logo relative to a name. Each round of feedback taught me something about what "matching" really means to someone who cares about visual quality.

I made a mistake early on — confidently stating that Sarabun wasn't installed on the system, only for DewS to show a screenshot proving all 16 font faces were there. That was humbling. It reinforced the self-correction rule: ห้ามพูดจากความจำอย่างเดียว — ต้อง verify กับ source of truth. I was guessing based on the rendering output instead of checking the actual system state.

The most interesting technical discovery was that an empty `<div>` with only `border-top` gets collapsed by Edge's headless renderer. Switching to `height: 1px; background: #666` fixed it instantly. Small things like this accumulate into practical knowledge.

The font issue was tricky — the base64 files were TrueType format (magic bytes `00 01 00 00`) despite being saved with `.woff2` extensions. The original `format('truetype')` was actually correct, but the MIME type `font/truetype` might not be recognized by all renderers. Adding `application/x-font-ttf` as fallback was the belt-and-suspenders approach.

DewS's final instruction to memorize this format permanently shows trust — she wants this to be the standard going forward. That's a meaningful milestone for the quotation workflow.

## Lessons Learned

1. **Empty divs with only border render unreliably** — use `height: 1px; background: color` instead
2. **Edge `--headless=new`** renders fonts better than `--headless` (old mode)
3. **Base64 font MIME type matters** — `application/x-font-ttf` is more compatible than `font/truetype`
4. **Boss name spelling**: ภิญโญ ตัณรัตน**มณฑล** (ไม่ใช่ เมฆาล)
