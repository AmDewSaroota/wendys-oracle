## Honest Feedback

**1. เข้าใจ previz direction ผิดหมด (Shot 4)** — นี่คือ friction ที่ใหญ่ที่สุดในเซสชันค่ะ ฉันเขียนให้มังกรบินเข้าหากล้อง ทั้งที่ต้องบินออกไป, เขียนกล้องอยู่ที่พื้น ทั้งที่อยู่เหนือเมฆ, เขียนกล้องแพนตาม ทั้งที่กล้องนิ่ง **ผิดทุกข้อ** เหตุผลคือฉันไม่ได้วิเคราะห์ previz frames อย่างถี่ถ้วน ถ้าฉันสังเกตว่ามังกรเล็กลงเรื่อยๆ ก็จะรู้ทันทีว่าบินออก ไม่ใช่บินเข้า ต้องฝึกให้ "ดูภาพ" จริงๆ ก่อนเขียน prompt ไม่ใช่คิดเอาเอง

**2. Neck restriction ทำลายทั้ง shot** — ฉันไม่ได้คิดให้รอบคอบพอก่อนเพิ่ม "保持基本朝前" Seedance ไม่ใช่ character rig ที่ control ได้ทีละ joint — มัน interprets ทั้ง prompt เป็นก้อนเดียว คำว่า "保持" (maintain/keep) + "朝前" (forward) ถูกตีความว่า "อย่าเคลื่อนไหว" ทั้งตัว ฉันควรเตือน DewS ว่า Seedance ไม่รองรับ fine-grained body part control แทนที่จะลองเพิ่มบรรทัดเข้าไปตรงๆ

**3. เสียเวลามากกับ Gemini ที่ทำไม่ได้** — เราลองให้ Gemini inpaint เมืองลงใน previz หลายรอบ ทุกรอบล้มเหลว ฉันควรจะ flag ตั้งแต่ต้นว่า Gemini มีข้อจำกัดเรื่อง layout preservation — จากประสบการณ์ที่เห็นผลลัพธ์ 2 รอบแรกก็ควรบอก DewS ได้แล้วว่า "Gemini ไม่เหมาะกับ task นี้ ควร composite เอง" แทนที่จะลองต่ออีกหลายรอบ

## Lessons Learned

1. **Seedance restriction words = global effect** — คำว่า "保持朝前" ไม่ได้แค่ lock หัว แต่ lock ทั้งร่าง Seedance ไม่มี body part isolation → อย่าใส่ restriction ถ้าไม่พร้อมเสียทั้ง shot
2. **Always study previz frames before writing** — ดูว่าวัตถุใหญ่ขึ้นหรือเล็กลง = บินเข้าหรือบินออก ตรวจ perspective ทุก frame ก่อนเขียนแม้แต่คำเดียว
3. **Previz video + simple prompt > complex text-only** — ปล่อยให้ video ref handle camera motion, text handle style/atmosphere เท่านั้น
4. **Know tool limits early** — ถ้า Gemini ล้มเหลว 2 รอบด้วยสาเหตุเดิม ให้หยุดแล้วเสนอทางอื่น อย่าลองซ้ำแบบเดิม
5. **Ending frame = teleport** — กฎเหล็กของ Seedance: ใช้แค่ starting frame + text, อย่าใช้ ending frame
