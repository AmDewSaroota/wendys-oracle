## Honest Feedback

**1. Context window exhaustion on iterative work.** This session spanned 2 context windows because HTML report editing generates massive context. Each read of the full file + edit + re-read eats context fast. For future report sessions, reading only relevant sections would help extend the window.

**2. Color contrast issues are hard to catch without rendering.** The white-on-white problems (summary table, timeline table) happened because I was reasoning about CSS in my head rather than seeing the actual render. DewS had to send screenshots twice. I should proactively flag any dark-background sections where table text might clash.

**3. replace_all can create unintended side effects.** The "ส่ง cloud" → "ดึงค่าได้" replacement hit a VOC row that had "✅ ส่ง cloud ได้!" — creating "✅ ดึงค่าได้ ได้!" with duplicate text. Using targeted edits instead of blanket replace would have avoided this.

## Lessons Learned

- **Lead with the domain expert's priority, not technical excitement.** TVOC cloud is technically impressive but PM2.5 reliability is what matters for the research.
- **Tone calibration matters in technical reports.** A fixable issue (sensor stuck → calibrate) shouldn't read like a critical failure.
- **Responsive CSS should be mobile-first for reports shared via LINE.** Advisors read on phones — 1 column default, expand to 3 on desktop.
