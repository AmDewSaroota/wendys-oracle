## Honest Feedback

**Friction 1: Plan-to-reality gap** — Plan เขียนไว้เป็น "slides" ชัดเจนตั้งแต่ต้น แต่ DewS ไม่ได้ต้องการ slides จริงๆ เธอต้องการ "เนื้อหาที่สอนได้" ตรงนี้ควรถาม DewS ตั้งแต่ plan mode ว่า format ไหนที่จะใช้จริง — slideshow? document? หรือ both? แทนที่จะ assume ว่า "สไลด์" = presentation format

**Friction 2: Context window management** — Session นี้เริ่มจาก compacted context ซึ่งทำให้ต้องอ่าน summary ยาวมากก่อนเริ่มงาน แต่ summary ดีมากครอบคลุมทุก change ที่ทำ ปัญหาคือ session ก่อนทำหลายอย่างมาก (SmartLife logo + device naming + auto-derive + date grouping + plan) จนเต็ม context

**Friction 3: Double write** — เขียน training.html 2 ครั้ง (slideshow → document) เพราะไม่ได้ clarify format ก่อน ถ้าถามก่อน 1 คำถาม ("เอาแบบ slide กด next หรือเอกสารเลื่อนดู?") จะประหยัดเวลาไปเยอะ

## Lessons Learned

- **Ask format before building** — เมื่อ user บอกว่าอยากได้ "สไลด์" อาจไม่ได้หมายถึง presentation format จริงๆ ควรถาม: "เอาแบบ slide กด next? หรือเอกสารเลื่อนดู?"
- **Mobile-first for field teams** — ทีมที่ทำงานภาคสนาม (อาสาสมัคร, แอดมินลงพื้นที่) ต้องการเอกสารที่เปิดมือถือได้ง่าย ไม่ใช่ presentation ที่ต้องมีคอมเต็มจอ
