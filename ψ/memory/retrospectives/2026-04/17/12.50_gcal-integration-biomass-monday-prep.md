## Honest Feedback

**Friction 1: MCP restart confusion.** I wasted time suggesting VS Code UI approaches (Ctrl+Shift+P → "MCP: List Servers") that don't apply to Claude Code's MCP config in `.claude.json`. Should have gone straight to terminal `npx auth` which is the actual fix. The MCP server restart workflow between VS Code extension and Claude Code CLI is confusing.

**Friction 2: Stale memory about MT13W.** My MEMORY.md still had MT13W as the main sensor even though DewS had switched to MT15. I presented a checklist saying "กด calibrate (3 ครั้ง)" which is wrong for MT15 (it's 4 ครั้ง). If DewS hadn't caught this, the training materials could have been misleading. Need to be more diligent about verifying hardware details before presenting checklists.

**Friction 3: Overcautious about collection_periods.** I flagged `collection_periods` being empty as a problem with a red ❌, but DewS immediately pointed out the whole point of Monday is to train admins to set these up themselves. I should have considered the context — "ก็จะไปสอนแอดมินตั้งไง" — before alarming about empty tables.

## Lessons Learned

1. **Terminal auth > UI restart for MCP token issues** — `npx @cocal/google-calendar-mcp auth` is the fastest fix, no need to restart IDE
2. **MT15 is the main sensor now** — calibrate = 4 presses (not 3), display toggle = 3 presses, WiFi pair = 5 presses
3. **Consider training context before flagging empty data** — empty tables before a training session is expected, not an error
4. **Google OAuth "Test Mode" = 7-day token expiry** — publish app in Cloud Console for permanent tokens
