## Honest Feedback

**Friction 1: Too many pivots on browser automation.** I wasted ~15 minutes trying dev-browser approaches that were doomed to fail. Google explicitly blocks automated Chromium for sign-in. I should have recognized this immediately — it's a well-known limitation — and gone straight to gcloud CLI or guided DewS through the browser manually.

**Friction 2: The backslash escaping was avoidable.** When writing Windows paths in JSON via Node.js, I should always use forward slashes or properly handle escaping. The fact that the path silently became `C:UsersCPL.claudegcp-oauth.keys.json` could have caused a confusing error on restart if I hadn't caught it.

**Friction 3: MCP config learning curve.** I initially tried to add `mcpServers` to `settings.json` which doesn't support it, then used `claude mcp add` with `-e` flag in args instead of env. Took 3 attempts to get the config right. Should have read the schema more carefully first.

## Lessons Learned

1. **Google blocks automated browsers** — For any Google auth, always use gcloud CLI or guide user through their own browser. Never try Playwright/Puppeteer for Google Sign-in.
2. **Windows paths in JSON** — Always use forward slashes (`C:/Users/...`) to avoid backslash escaping issues.
3. **Tuya Trial can be extended** — Trial Edition can't be renewed but CAN be extended via "Extend Trial Period" form. Frame as individual/personal use.
