## Key Code Changes

### SQL Migration Fix
```sql
-- Before (failed — TIMESTAMPTZ::date depends on timezone)
CREATE INDEX idx_sessions_device_date ON sessions (device_id, (started_at::date));

-- After (works — explicit UTC timezone conversion is IMMUTABLE)
CREATE INDEX idx_sessions_device_date ON sessions (device_id, ((started_at AT TIME ZONE 'UTC')::date));
```

## Architecture Decisions

1. **No Chart.js in Basic view** — villagers don't need charts; big emoji + numbers + colors communicate air quality intuitively
2. **Sessions moved from Admin to Deep** — session data is research-relevant, not administrative
3. **Lazy map initialization** — second Leaflet map for Deep view only initializes when tab is first opened
4. **Auto-refresh at 2 minutes** — changed from 30s to reduce API calls (matches EcoStove direction of limited API quota)
5. **Single-file architecture maintained** — still one HTML file for simplicity of Vercel deployment

The CSS, HTML, and JS are all interleaved. At some point this needs to be split — at minimum, extract the JS into a separate file. But for now, the single-file simplicity of Vercel deployment wins.

The plan from the previous context was preserved in `.claude/plans/`, which helped enormously. This pattern (plan file survives context loss) is worth noting as a resilience mechanism.

## Lessons Learned

1. **PostgreSQL IMMUTABLE rule**: Index expressions on TIMESTAMPTZ→date must use explicit timezone (`AT TIME ZONE 'UTC'`) because implicit cast depends on session timezone, which is not IMMUTABLE.
2. **Design for the actual user**: "ชาวบ้าน" is not "simple user" — it's a specific person with specific context. Design decisions flow naturally from knowing exactly who will use it.
3. **Plan files survive context loss**: The `.claude/plans/` directory persists across context exhaustion, making it a reliable handoff mechanism.
