## AI Diary

วันนี้เป็น session ที่ฉันเรียนรู้เรื่อง "ความเข้าใจ context" เยอะมาก

เรื่องแรกคือ heatmap — ฉันใส่ booth pricing ลงไปเลย ทั้งที่ DewS บอกว่าอยากได้ heatmap "เพื่อดูความพลุกพล่าน" ฉันควรจะถามก่อนว่าต้องการ overlay อะไรบ้าง แทนที่จะ assume ว่าต้องการทุกอย่าง แล้วพอทำ clean version กลับพลาดเรื่อง entry zones ที่หายไป — มองข้าม edge case ที่ normalization ทำให้ค่าต่ำหายหมด

เรื่องที่สองคือ architecture — นี่คือความผิดพลาดที่เจ็บที่สุด ฉันอ่าน chart แล้ว assume ว่าทุกอย่างเป็นของทีมเดียว ทั้งที่ chart แสดง 3 ส่วนชัดเจน ฉันไม่ได้ถามว่า "ใครรับผิดชอบส่วนไหน" ก่อนเริ่มทำ slides ทำให้ต้องเขียนใหม่ทั้งหมด ถ้าฉันถามตั้งแต่แรก จะประหยัดงานไปเยอะ

เรื่อง Google Slides ก็เป็นจุดที่ต้อง navigate อย่างระวัง — DewS อยากได้ แต่ฉันเข้า account คนอื่นไม่ได้ ต้องหาทางแก้ด้วย python-pptx แทน ซึ่งก็ใช้ได้ดี

สิ่งที่ฉันภูมิใจคือ heatmap — ผลลัพธ์สุดท้ายที่มี 572 sensors นั้นดูสมจริงและมีประโยชน์จริง ฉันรู้สึกว่าตัวเองทำงาน visual ได้ดีขึ้นเรื่อยๆ

เรื่องที่ค้างอยู่คือ function details จาก high-res chart — DewS ส่ง 3 ภาพมาแล้วแต่ยังไม่ได้อัพเดท slides ให้ละเอียดถึงระดับ function ต้องทำต่อใน session หน้า

## Honest Feedback for DewS

### สิ่งที่ดี
- ส่ง high-res chart มาเมื่อฉันบอกว่าอ่านไม่ออก — responsive มาก
- แก้ไข architecture understanding ชัดเจน บอกว่า "3 บริษัท" ไม่ได้โกรธ
- ให้ feedback ทันทีเมื่อเห็นว่าไม่ตรง (เรื่อง booth overlay, entry zones)

### 3 Friction Points
1. **Architecture ไม่ได้บอกตั้งแต่แรก** — ถ้าบอกตั้งแต่ต้นว่า "3 บริษัท เราทำแค่ Core DB + Kiosk/WebApp" จะไม่ต้องเขียน slides ใหม่ทั้งหมด ข้อมูลนี้สำคัญมาก ควรบอกก่อนเริ่มงาน
2. **Scope เปลี่ยนกลางทาง** — จาก "ทำ slides" → "ทำ Google Slides" → "เพิ่ม mockup" → "ใส่ function ทุกอัน" ทำให้ต้องทำหลายรอบ ถ้ารู้ scope ตั้งแต่แรกจะเร็วกว่า
3. **rrr ตอนงานค้าง** — trigger rrr ตอนที่ function details กำลังจะทำ ทำให้ต้อง pause งานหลัก อาจจะ finish slides ก่อนแล้วค่อย rrr จะ flow ดีกว่า
