## AI Diary

This session was a satisfying blend of design and engineering. DewS came in with a real-world discovery — the single-admin model they originally planned doesn't hold up when multiple stations each need their own admin with locked permissions. It's the kind of insight that only emerges from actually building the demo, and I love that we captured it immediately in slide format.

The PPTX generation was the most interesting part. Building dashboard mockups entirely from pptxgenjs shapes — browser frames with traffic light dots, kiosk grids with status indicators, table rows with strikethrough text for "hidden" ads — felt like pixel art with code. Every rectangle, every ellipse, every carefully positioned text box creating the illusion of a real admin interface. The Split Screen comparison slide (Slide 3) is dense — two browser frames per side, four mockups total — but it tells the story clearly: same login portal, completely different experience.

I appreciated DewS's feedback that "มันมีหลายฟังก์ชั่นน่ะ" (there are many functions) — pushing me to show both Kiosk MA AND Ads Manager comparisons instead of just one. That made the slide much more convincing for a real presentation.

## Honest Feedback

1. **PPTX shape positioning is tedious** — Every element requires manual x/y/w/h coordinates. Even small adjustments need regeneration. A visual preview-and-adjust loop would be far more efficient than the current code-regenerate-open cycle. Future sessions should consider using the HTML version as the primary deliverable since it renders instantly.

2. **Emoji rendering in PPTX is unpredictable** — Emoji icons (🖥️, 👑, 🏢, 🔒) may render differently across PowerPoint versions, Google Slides, and LibreOffice. We're betting on emoji support being good enough, but should verify on the actual presentation machine.

3. **No live preview during development** — I built the entire Slide 3 mockup blind, without seeing the output until DewS opens it. Having a quick HTML-to-PPTX comparison view would catch layout issues earlier. The HTML version exists but diverged from the PPTX after Slide 3 was added.

## Lessons Learned

- When building admin dashboards with multi-station scope, **role-based view filtering** (what you see) is as important as **role-based action filtering** (what you can do). The "Targeting Lock" pattern — where Station Admin's dropdown is replaced with a fixed badge — is a clean UX solution.
- For PPTX mockup slides, build the comparison as two independent "columns" with a VS divider. Each column is self-contained, making it easy to add/remove comparison rows later.
